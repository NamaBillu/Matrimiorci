<task type="auto">
  <name>Task 1: Create SetAudioCompression Editor script</name>
  <files>Assets/Editor/SetAudioCompression.cs</files>
  <action>
Create `Assets/Editor/SetAudioCompression.cs`. This is a Unity Editor utility script (not compiled at runtime) that batch-sets all AudioClips in the project to CompressedInMemory load type.

Requirements:
- Class name: `SetAudioCompression`
- Namespace: none (matches existing Editor conventions — see `Assets/Editor/`)
- MenuItem path: `"Tools/Set All Audio to CompressedInMemory"`
- Uses `AssetDatabase.FindAssets("t:AudioClip")` to find all clips
- For each clip: gets `AudioImporter`, reads `defaultSampleSettings`, sets `loadType = AudioClipLoadType.CompressedInMemory`, writes back with `SaveAndReimport()`
- Logs each processed path with `Debug.Log`
- Logs a summary count at the end: "Done: set {N} AudioClips to CompressedInMemory"
- Does NOT change `compressionFormat` or `quality` — only `loadType`
- Must also apply to platform-specific overrides for iOS and Android (set `overrideSampleSettings` for those platforms if already overridden, otherwise only default is sufficient)

Note: If no AudioClips exist yet when the script is run, it logs "No AudioClips found." and exits gracefully. This is expected — audio assets will be added by the developer and the tool can be re-run.
  </action>
  <verify>
    <automated>Get-ChildItem "Assets/Editor/SetAudioCompression.cs" | Select-Object Name, Length</automated>
  </verify>
  <done>File exists. Unity Editor compiles it without errors (verify in Unity Console after file creation).</done>
</task>

<task type="checkpoint:human-action">
  <what-built>Editor script for audio compression is ready. Now the developer must apply Unity Player Settings for the WebGL production build.</what-built>
  <how-to-verify>
Open Unity Editor. Apply each setting exactly as listed:

**1. Switch to WebGL platform (if not already):**
File → Build Settings → Select WebGL → "Switch Platform" button

**2. Open Player Settings:**
File → Build Settings → "Player Settings..." button (or Edit → Project Settings → Player)

**3. WebGL tab — Publishing Settings:**
- Compression Format: **Gzip** ← critical, do not use Brotli

**4. WebGL tab — Memory:**
- Initial Memory Size (MB): **32**
  (Verify it's not set to a higher value)

**5. WebGL tab — Other Settings (WebAssembly):**
- Target: **WebAssembly 2023** ← must be checked
  (BigInt is automatically included with WebAssembly 2023)

**6. Other Settings — Code Stripping:**
- Enable Exceptions: **None** (reduces WASM size)
- Managed Stripping Level: **Medium**

**7. Other Settings — Logging:**
- Stack Trace: set all to **None** for production

**8. Run the audio compression tool:**
Tools menu → **Set All Audio to CompressedInMemory**
Check Unity Console — it should log each processed clip (or "No AudioClips found." if audio hasn't been added yet).

**9. Verify DOTween link.xml exists:**
Check that `Assets/Plugins/Demigiant/DOTween/` contains a `link.xml` file (prevents stripping of DOTween reflection code). If missing, this will cause runtime NullRefs with code stripping enabled.
  </how-to-verify>
  <resume-signal>Type "settings applied" when all Player Settings are configured and the audio tool has been run.</resume-signal>
</task>
