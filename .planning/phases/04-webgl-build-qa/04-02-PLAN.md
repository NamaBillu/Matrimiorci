  <how-to-verify>
**Step 1 — Run the Unity WebGL build:**

1. File → Build Settings → confirm WebGL is selected platform
2. Click **"Build"** (not "Build and Run")
3. Create output folder when prompted: name it `WebGLBuild` (outside the Unity project root if possible, to avoid Unity re-importing it)
4. Wait for build to complete — watch Console for errors
5. After build: note the **total uncompressed size** printed in Console
6. Check compressed sizes in `WebGLBuild/Build/` folder:
   - `*.data.gz` + `*.wasm.gz` + `*.framework.js.gz` combined **must be ≤ 12MB**
   - If over 12MB: increase Managed Stripping Level to High in Player Settings and rebuild

**Step 2 — Set up gh-pages branch:**

Run these git commands (or use GitHub Desktop):

```bash
# From the repo root — create orphan gh-pages branch
git checkout --orphan gh-pages

# Remove all tracked files from this branch
git rm -rf .

# Copy build output into the branch root
# (adjust path if you built outside the repo)
Copy-Item -Recurse WebGLBuild\* .

# Add .nojekyll (already in working tree from Task 1)
# If it got removed by git rm, recreate it:
New-Item -ItemType File -Name ".nojekyll" -Force

# Commit and push
git add -A
git commit -m "deploy: WebGL build to gh-pages"
git push origin gh-pages --force

# Return to main branch
git checkout main
```

**Step 3 — Configure GitHub Pages:**

1. Open the repository on GitHub.com
2. Settings → Pages
3. Source: **Deploy from a branch**
4. Branch: **gh-pages** / folder: **/ (root)**
5. Click Save
6. Wait ~60 seconds for Pages to publish
7. Visit the published URL (shown in Settings → Pages): `https://{user}.github.io/{repo}/`
8. Confirm the app loads (Unity loading bar appears, then content)

**Step 4 — Verify QR URL:**

Manually visit: `https://{user}.github.io/{repo}/?type=invite`

Confirm the **Invite scene** loads (not Home scene directly).

Visit: `https://{user}.github.io/{repo}/`

Confirm the **Home scene** loads by default.
  </how-to-verify>
  <resume-signal>Type "deployed" when the app is live on GitHub Pages and both URL variants (invite + default) load correctly.</resume-signal>
