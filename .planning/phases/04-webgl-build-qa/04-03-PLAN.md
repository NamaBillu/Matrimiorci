<task type="checkpoint:human-verify">
  <what-built>The app is deployed. Run QA on Windows browsers first (fastest iteration loop for fixing bugs).</what-built>
  <how-to-verify>
Open `https://{user}.github.io/{repo}/` in each of Chrome, Firefox, Edge.

Work through this checklist for each browser. Mark ✅ or ❌:

**Loading**
- [ ] App loads without console errors (F12 → Console)
- [ ] Unity loading bar appears and completes
- [ ] No "decompress" errors in console (would indicate Gzip/Brotli mismatch)

**URL Routing**
- [ ] `?type=invite` → Invite scene loads
- [ ] No param → Home scene loads
- [ ] Returning to the page after entering a code → Home scene loads (ROUT-02)

**Invite Scene**
- [ ] Invitation displayed correctly (fonts, colors, wedding image)
- [ ] "Entra" / open invitation button navigates to Home

**Password Flow**
- [ ] Code entry popup is accessible
- [ ] Valid code unlocks content, persists on page refresh
- [ ] Invalid code shows Italian error message
- [ ] No console errors during unlock

**Home Scene — Content**
- [ ] Wedding date, time, venue, address displayed
- [ ] Google Maps link opens correctly in new tab
- [ ] Schedule/timeline shown
- [ ] Dress code shown
- [ ] FAQ (≥5 questions) shown
- [ ] Contact info shown
- [ ] Accommodations section shows lock overlay until code entered

**Personalized Section (after unlock)**
- [ ] Guest group names displayed
- [ ] Apartment-group groups: breakfast note/field visible
- [ ] Non-apartment groups: breakfast field NOT visible

**RSVP**
- [ ] RSVP accessible only after code entry
- [ ] Guest names pre-filled (one row per group member)
- [ ] Attendance toggles (yes/no) work per guest
- [ ] Meal preference toggles work per guest
- [ ] Notes/dietary field accepts text input
- [ ] Submit: button disables + spinner shows
- [ ] Submit: confirmation appears on success (check Google Sheet for new row)
- [ ] Re-visiting RSVP after submission → shows confirmation (no form)

**Sound**
- [ ] Sound effects play (if any audio added)
  </how-to-verify>
  <resume-signal>Type "windows pass" if all checks pass, or describe any failures found (e.g., "RSVP spinner doesn't disappear — Firefox only").</resume-signal>

**Android (physical device — Chrome):**

Open `https://{user}.github.io/{repo}/` in Chrome on the Android device.

- [ ] App loads — Unity loading bar appears and completes
- [ ] QR code test: scan a QR pointing to `?type=invite` → Invite scene loads
- [ ] Touch gestures work (tap buttons, scroll inside popups)
- [ ] Password keyboard appears on tapping code input field
- [ ] Valid code entered → content unlocked
- [ ] RSVP popup opens — all input fields usable on touch keyboard
- [ ] Submit RSVP → success confirmation
- [ ] Audio plays correctly (if any clips added)

---

**iOS (physical iPhone — Safari):**

Open `https://{user}.github.io/{repo}/` in Safari on the iPhone.

- [ ] App loads — Unity loading bar appears and completes
- [ ] **Safe area check:** Content is not hidden behind notch or home bar indicator (especially on iPhone with Dynamic Island or notch)
- [ ] QR code test: scan QR with iPhone camera → link opens in Safari → Invite scene loads
- [ ] Touch input works throughout
- [ ] Password keyboard appears on code input tap
- [ ] Valid code entered → content unlocked
- [ ] RSVP popup — keyboard appears for text fields, form is fully scrollable
- [ ] Submit RSVP → success
- [ ] **Silent mode audio test:** Toggle iPhone to Silent Mode (hardware switch) → app should still load without crash; audio may be muted (acceptable — iOS behavior)
- [ ] **Audio post-tap test:** Tap anywhere on the app → after first tap, any sound effects should play (iOS AudioContext unlock)
- [ ] Background and return: press Home button, return to Safari tab → app should resume (may reload — acceptable)
